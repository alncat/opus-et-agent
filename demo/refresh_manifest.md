## Gated regen sequence

### S0 — Expanded training completes (gate)
- **Wait for:** job **160089** → `opuset/ribo/z8_expanded/weights.35.pkl` (10 warm epochs from `weights.25`, NUM_EPOCHS=36 → final epoch 35). Monitor: `bjgj9hmde`.
- **Verify:** `ls opuset/ribo/z8_expanded/weights.35.pkl config.pkl` and confirm `config.pkl` shows `zdim=8, z_affine_dim=6, angpix=4.2` (must match the baseline it warm-started from).

### S1 — Analyze the new latent
```bash
sbatch --export=ALL,SKILL_DIR=$SKILL scripts/analyze_opuset.slurm   # z8_expanded auto-read from config.pkl
# → opuset/ribo/z8_expanded/analyze.35/kmeans20/{labels.pkl, *.mrc}, umap.pkl, pca.pkl
```
**Re-identify the ribosome states** (Gate-3 judgment on the NEW cluster IDs — they renumber
every run; do not assume k6/8/9/10 carries over). This is the human checkpoint, informed by S2.

### S2 — Gate-3 signal assets (all read `analyze.35/kmeans20`)
```bash
# template correlation montage + scores
python opus-et-analysis/scripts/compare_to_template.py \
    --states-dir opuset/ribo/z8_expanded/analyze.35/kmeans20 \
    --template templates/ribo_tm.mrc --apix 4.2 -o gate3/states_vs_template
# N×N map-to-map consistency (raw + lp30)
python opus-et-analysis/scripts/state_consistency.py \
    --states-dir opuset/ribo/z8_expanded/analyze.35/kmeans20 --apix 4.2 -o gate3/state_consistency
# per-tomogram stats (need labels.pkl + the star's tomogram column + chosen clusters)
python opus-et-analysis/scripts/state_tomo_stats.py \
    --labels opuset/ribo/z8_expanded/analyze.35/kmeans20/labels.pkl \
    --star warp_tiltseries/ribo_matching.star --selected <NEW_CLUSTER_IDS> -o gate3/tomo_stats
```
- **3D gallery** (ad-hoc): open all `kmeans20/*.mrc` in ChimeraX, fair-color each, `tile`,
  `lighting soft`, save `ribo_state_gallery_3d.png` + the `view_states.cxc` bundle.
- **UMAP-by-state** (ad-hoc): load `umap.pkl` + `labels.pkl`, scatter colored by k-means
  label → `ribo_latent_umap_states.png`.
> Confirm the flag args (`--states-dir`, `--template`, `-o`) against each script's `--help`
> before running — the exact flag names are the source of truth, not this manifest.

### S3 — Build the selection star
```bash
sbatch --export=ALL,SKILL_DIR=$SKILL,SELECT_CLUSTERS="<NEW_CLUSTER_IDS>" scripts/select_state.slurm
# → warp_tiltseries/sel_ribo.star (OPUS-ET-refined poses injected, -D 104 --Apix 4.2)
```

### S4 — Fixed-mode half-maps + corrected FSC (Gate 4)
```bash
# split sel_ribo.star into rlnRandomSubset halves, then:
sbatch --export=ALL,SKILL_DIR=$SKILL,FIXED_SUBSET_LABEL=1 scripts/train_opuset_fixed.slurm
sbatch --export=ALL,SKILL_DIR=$SKILL,FIXED_SUBSET_LABEL=2 scripts/train_opuset_fixed.slurm
# molecule mask from the density (also the M_MASK):
python opus-et-analysis/scripts/gen_mask_from_map.py \
    --half1 opuset/ribo/fixed_subset1/tmp0.mrc --half2 opuset/ribo/fixed_subset2/tmp0.mrc -o mask.mrc
# gold-standard FSC WITH phase-randomization correction (default on):
python opus-et-analysis/scripts/compute_fsc.py \
    --half1 opuset/ribo/fixed_subset1/tmp0.mrc --half2 opuset/ribo/fixed_subset2/tmp0.mrc \
    --apix 4.2 --mask mask.mrc -o gate3/fsc_ribo
# → gate3/fsc_ribo_fsc.png (masked + randomized + corrected overlay) + _fsc.tsv
```

### S5 — M refinement
- `warp_m_*` (advanced.md) with half1/half2 + `mask.mrc` + `sel_ribo.star` →
  refined map + `ribo_matching_refined.star`.

### S6 — In-cell ArtiaX finale
- `opus-et-visualize/scripts/gen_artiax_scene.py` on the refined poses → in-cell scene;
  render still/short orbit for Scene 5.

### S7 — Pull assets down + update narrative
```bash
# from the local repo:
scp super:$WORK/gate3/*.png super:$WORK/gate3/*.tsv demo/qc/gate3_states/
scp super:$WORK/opuset/ribo/gate3/fsc_ribo_fsc.* demo/qc/gate3_states/
```
Then update the **numbers** in [`README.md`](README.md) Gate-3 section and
[`video_script.md`](video_script.md) Scene 4/5:
- subtomo count 50k → **58k**, epochs, cluster IDs of the high-res core, particle counts,
  mutual-CC values, UMAP island fractions;
- corrected FSC resolution (replace the earlier masked-only number);
- remove the ⚠️ "pending refresh" callouts once done.
